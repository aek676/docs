Esta actividad consiste en crear una herramienta para que los usuarios informen de incidencias (desperfectos, problemas de mantenimiento, etc.) capturando evidencia visual y geográfica. Se utilizan plugins nativos de Capacitor para acceder a la cámara y a la geolocalización del dispositivo, y se almacenan los datos de forma persistente con localStorage.

---

## 1. Preparación del entorno

Se añaden las dependencias de Capacitor para las plataformas nativas y los plugins de geolocalización:

```bash
npm install @capacitor/android
npm install @capacitor/ios
npm install @capacitor/geolocation
```

Se actualiza la configuración de Capacitor en `capacitor.config.ts` con el `appId` y `appName` del proyecto, y se excluyen los directorios `android/` e `ios/` del control de versiones.

<Aside type="note">

Desde hace un par de versiones, Ionic adoptó Capacitor como su motor por defecto para aplicaciones nativas, reemplazando al antiguo Cordova.

</Aside>

---

## 2. Geolocalización del dispositivo

Para obtener la geolocalización del dispositivo, se instala el plugin correspondiente y se sincroniza:

```bash
npm install @capacitor/geolocation
npx cap sync
```

El objetivo de `npx cap sync` es sincronizar los archivos de origen del proyecto con los archivos nativos de la plataforma objetivo (como iOS y Android). Esto puede incluir la configuración de permisos necesarios en `Info.plist` para iOS o `AndroidManifest.xml` para Android.

En un primer paso, se implementa la geolocalización directamente en la página principal, mostrando latitud, longitud y precisión en una tarjeta `ion-card`.

<Aside type="note">

El método `Capacitor.isNativePlatform()` permite detectar si la aplicación se ejecuta en un dispositivo nativo (iOS/Android) o en un navegador. En plataformas nativas se usa `Geolocation.getCurrentPosition()` de Capacitor; en web se utiliza `navigator.geolocation.getCurrentPosition()` como fallback.

Para utilizar la cámara del dispositivo, se instalan las dependencias necesarias:

<Aside type="note">

Se utiliza `CameraResultType.Uri` para obtener la imagen como una URL de archivo en lugar de Base64. La comunicación entre la WebView y el Hardware se realiza a través de un Native Bridge donde toda información debe ser serializada en JSON. Las cadenas Base64 de imágenes de alta resolución pueden saturar la memoria de la WebView, por lo que siempre se priorizan las URLs de archivo.

</Aside>

### Métodos implementados

1. **`getAll()`**: Recupera todas las incidencias almacenadas en `localStorage` bajo la clave `incidences`
2. **`create(incidence)`**: Añade una nueva incidencia al array existente y lo guarda

<Aside type="note">

### Flujo de captura

1. **Placeholder inicial**: Hasta que no se obtenga ninguna foto, se muestra un icono de cámara y un mensaje indicando que no se ha capturado ninguna foto
2. **Captura simultánea**: Se utiliza `Promise.all` para ejecutar en paralelo la captura de foto (`CameraService`) y la obtención de geolocalización (`GeolocationService`)
3. **Preview**: Tras la captura, se muestra la foto y debajo las coordenadas (latitud y longitud)
4. **Guardado**: Al pulsar "Guardar Incidencia", se crea un objeto `Incidence` y se persiste con `IncidenceService`
5. **Navegación automática**: Tras guardar, se redirige al listado de incidencias
6. **Feedback visual**: Se utilizan `ToastController` para mostrar mensajes de éxito o error
7. **Indicadores de carga**: Se muestran spinners mientras se captura o se guarda

</Aside>

## 9. Ejecución del proyecto

<Steps>

1. Instalar dependencias

   ```bash
   npm install
   ```

2. Ejecutar en el navegador

   ```bash
   ionic serve
   ```

3. Esto abrirá la aplicación en el navegador por defecto en `http://localhost:8100`.

</Steps>

<Aside type="tip">

Para probar en dispositivos nativos, es necesario añadir las plataformas y ejecutar:

```bash
npx cap add android
npx cap add ios
ionic build
npx cap sync
npx cap open android
npx cap open ios
```

</Aside>
