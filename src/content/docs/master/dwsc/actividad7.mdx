El objetivo de esta actividad es desarrollar una arquitectura de microservicios que:

- Implemente dos servicios independientes (OMDB y TMDB) para buscar películas utilizando sus APIs externas
- Agregue los servicios mediante OpenFeign en un movies-manager
- Utilice Eureka Server para el descubrimiento de servicios
- Configure Spring Cloud Gateway como punto de entrada único

## 1. Arquitectura del Proyecto

El proyecto está formado por los siguientes componentes:

```d2 sketch
direction: down

infraestructura: {
  eureka: Eureka Server {
    shape: cylinder
  }

  gateway: Spring Cloud Gateway {
    shape: hexagon
  }
}

microservicios: {
  manager: Movies Manager
  omdb_service: OMDB Service
  tmdb_service: TMDB Service
}

apis_externas: {
  omdb: API OMDB {
    shape: cloud
  }

  tmdb: API TMDB {
    shape: cloud
  }
}

usuario: {
  shape: person
  width: 130
}

# --- Flujo de la Petición Principal ---
usuario -> infraestructura.gateway: Petición HTTP
infraestructura.gateway -> microservicios.manager: Ruteo

microservicios.manager -> microservicios.omdb_service: OpenFeign
microservicios.manager -> microservicios.tmdb_service: OpenFeign

microservicios.omdb_service -> apis_externas.omdb: REST
microservicios.tmdb_service -> apis_externas.tmdb: REST

# --- Flujo de Descubrimiento de Servicios ---
# Utilizando stroke-dash como en la documentación para diferenciar estas líneas

infraestructura.gateway -> infraestructura.eureka: Discovery {
  style.stroke-dash: 5
}

microservicios.manager -> infraestructura.eureka: Discovery {
  style.stroke-dash: 5
}

microservicios.omdb_service -> infraestructura.eureka: Discovery {
  style.stroke-dash: 5
}

microservicios.tmdb_service -> infraestructura.eureka: Discovery {
  style.stroke-dash: 5
}
```

<Aside type="note">

La arquitectura sigue el patrón de arquitectura de microservicios donde:

- **Eureka Server** permite el descubrimiento dinámico de servicios
- **OMDB** y **TMDB** son servicios independientes que consultan APIs externas
- **Movies Manager** agrega las respuestas de ambos servicios mediante Feign
- **Gateway** actúa como punto de entrada único para todas las peticiones

</Aside>

## 9. Ejecución del proyecto

<Steps>

1. **Generar las imágenes Docker**

   Para cada microservicio, generar la imagen:

   ```bash
   cd eureka-server
   ./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ghcr.io/aek676/eureka-server:latest

   cd omdb-service
   ./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ghcr.io/aek676/omdb-service:latest

   cd tmdb-service
   ./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ghcr.io/aek676/tmdb-service:latest

   cd movies-manager
   ./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ghcr.io/aek676/movies-manager:latest

   cd gateway
   ./mvnw spring-boot:build-image -Dspring-boot.build-image.imageName=ghcr.io/aek676/gateway:latest
   ```

2. **Crear archivo .env**

   Crear un archivo `.env` con las claves de API:

   ```env
   OMDB_API_KEY=tu_api_key_de_omdb
   TMDB_API_KEY=tu_api_key_de_tmdb
   ```

3. **Ejecutar con Docker Compose**

   ```bash
   docker-compose up -d
   ```

4. **Verificar los servicios**
   - Eureka Server: [`http://localhost:8761/`](http://localhost:8761/)
   - Gateway: [`http://localhost:8080/movies-manager/movie?title=inception`](http://localhost:8080/movies-manager/movie?title=inception)

</Steps>
