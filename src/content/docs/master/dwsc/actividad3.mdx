El objetivo de esta actividad es implementar un servicio RESTful utilizando
Java, Jersey y WildFly. El servicio permitirá leer y añadir usuarios dando
como resultado un JSON o un XML (legacy).

## 1. Inicialización del proyecto inicial

Para desarrollar la actividad se va a utilizar un proyecto Maven,
además de configurar Jersey para crear el servicio RESTful y WildFly
para el servidor del servicio. Procedemos a explicar la actividad:

<Steps>

1. Inicialización del proyecto Maven:
  
    Se ha inicializado un proyecto Maven con utilizando el siguiente comando con packaging war:

    ```bash
    mvn archetype:generate -DgroupId=es.ual -DartifactId=actividad3 -DarchetypeArtifactId=maven-archetype-webapp -Dpackaging=war -DinteractiveMode=false
    ```

2. Modificar web.xml:

    Tenemos que modificar el acrhivo `web.xml` para definiar el servlet
    de Jersey, además de configurar el mapeo a `/api/*` y indicar donde

   <RemoteCode
     url={`${raw}src/main/webapp/WEB-INF/web.xml`}
     lang="xml"
     title="src/app/main/webapp/WEB-INF/web.xml"
   />

3. Modificar pom.xml:

    Para incluir las dependencias necesarias para el proyecto, se ha modificado
    el archivo `pom.xml` para incluir las dependencias que se van a necesitar.

    <RemoteCode
     url={`${raw}pom.xml`}
     lang="xml"
     title="pom.xml"
    />

</Steps>
