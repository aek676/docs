El objetivo de esta actividad es desarrollar una arquitectura de microservicios utilizando
**Spring Boot** con **Spring Cloud Config** para la configuración centralizada.
Se implementan dos microservicios: uno que utiliza JDBC puro y otro que utiliza
JPA con relaciones ManyToMany.

## 1. Arquitectura del Proyecto

El proyecto está formado por los siguientes componentes:

| Componente | Puerto | Tecnología |
|------------|--------|-------------|
| Config Server | 8888 | Spring Cloud Config |
| Manager Users | 8081 | JDBC puro |
| Manager Students | 8082 | JPA + Spring Data |

<Aside type="note">

La configuración centralizada permite gestionar los parámetros de todos
los microservicios desde un repositorio Git externo, facilitando el mantenimiento
y los despliegues.

</Aside>

## 6. Ejecución del proyecto

<Steps>

1. **Iniciar las bases de datos**

   ```bash
   docker-compose up -d
   ```

2. **Compilar el Config Server**

   ```bash
   cd config-server
   ./mvnw clean package -DskipTests
   ```

3. **Compilar Manager Users**

   ```bash
   cd manager-users
   ./mvnw clean package -DskipTests
   ```

4. **Compilar Manager Students**

   ```bash
   cd manager-students
   ./mvnw clean package -DskipTests
   ```

5. **Ejecutar los servicios**

   Ejecutar en terminals separadas:

   ```bash
   # Terminal 1: Config Server
   cd config-server
   ./mvnw spring-boot:run

   # Terminal 2: Manager Users
   cd manager-users
   ./mvnw spring-boot:run

   # Terminal 3: Manager Students
   cd manager-students
   ./mvnw spring-boot:run
   ```

6. **Verificar los servicios**

   - Config Server: [`http://localhost:8888/actuator/health`](http://localhost:8888/actuator/health)
   - Manager Users: [`http://localhost:8081/swagger-ui.html`](http://localhost:8081/swagger-ui.html)
   - Manager Students: [`http://localhost:8082/swagger-ui.html`](http://localhost:8082/swagger-ui.html)

</Steps>
